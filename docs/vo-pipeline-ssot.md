# 游戏语音管线 SSOT 设计脑暴

## 核心思路

以一条 **VO Line** 为最小单位，向外辐射整个语音管线的复杂度。VO Line 是整个语音管线的 **SSOT（Single Source of Truth）**——所有上下游角色以它为中心协作。

## 聚焦范围

以 **Shared Slot**（通用槽位）类型的 VO Line 为主线展开讨论。Shared Slot 是所有角色都需要准备的通用语音，如：战斗时使用药品、受伤、低血量、死亡等。

剧情 / 过场动画类 VO 暂不展开。

---

## VO Line 结构

### 核心字段

| 字段 | 说明 |
|---|---|
| **Character** | 角色标识，确定谁在说话 |
| **Text** | 台词文本内容 |
| **Shared Slot** | 通用槽位分类（死亡 / 受伤 / 低血量 / 用药 等） |
| **VO Perspective** | VO_1P / VO_3P / VO_1P_Critical，同时表达**视角区分**和**播放优先级** |

> 每条 VO Line 不直接引用 .wav 文件，而是引用音频中间件资产（如 Wwise Event 或 External Source）。

### 多语种资产

同一条 VO Line 下挂载多语种版本，由运行时语言设置切换：

| 语种 | 资产引用 |
|---|---|
| CN | → Wwise Event / External Source |
| JP | → Wwise Event / External Source |
| EN | → Wwise Event / External Source |
| KR | → Wwise Event / External Source |

> 后续可扩展更多语种。

## 上游流程链

从需求到资产交付的完整链路：

```
本地化 (Localization) → 选角试音 (Casting) → 配音录制 (Recording) → 后期处理 (Post-processing)
```

| 环节 | 涉及角色 |
|---|---|
| 本地化 | 翻译 |
| 选角试音 | 配音导演 |
| 配音录制 | 配音演员、录音师 |
| 后期处理 | 音频设计师 |

---

## 核心协作角色

四个核心角色以 VO Line 为协作界面：

```
策划 (Designer) → 文案 (Writer) → 音频 (Audio) → 程序 (Programmer)
```

| 角色 | 职责 |
|---|---|
| **策划** | 提出需求（如"角色死亡时要说一句话"） |
| **文案** | 撰写具体台词内容 |
| **音频** | 提供声音资产 + 配置声音的传播方式和行为 |
| **程序** | 具体的触发实现 |

---

## 触发模块

同一条 Shared Slot VO Line（如"死亡"）会被多个游戏模块在不同场景下触发：

- **战斗系统**：玩家击杀敌人时触发
- **AI 系统**：AI 死亡时触发
- **UI 系统**：UI 提示语音
- **背包系统**：物品使用语音
- **技能系统**：角色释放技能时的喊话

> 触发逻辑由程序在各模块中管理，VO Line 不携带触发条件字段。

---

## 生命周期状态

一条 VO Line 从空缺到上线的完整状态流转：

```
空缺 → TTS（占位） → 待录音 → 已录音 → 已处理 → 已集成 → 已验收
```

| 状态 | 说明 |
|---|---|
| 空缺 | 需求已确认但尚无语音 |
| TTS | 使用 TTS 生成临时占位语音，用于快速原型验证 |
| 待录音 | 已安排录音但尚未录制 |
| 已录音 | 已完成棚录，等待后期 |
| 已处理 | 已完成后期处理（降噪、EQ、响度标准化等） |
| 已集成 | 已导入音频中间件 / 引擎并配置完成 |
| 已验收 | 最终审核通过 |

---

## 跨资产关联

VO Line 作为 SSOT，与以下资产类型存在关联：

| 资产类型 | 关系 |
|---|---|
| **字幕 / UI** | VO 触发时同步显示字幕组件或 UI 文本 |
| **面部动画 / 口型** | 动画系统消费 VO，驱动面部表情和口型同步 |

> 动画 / 口型数据不在 VO Line 的字段数据库中。VO Line 只负责提供音频侧的确定性信息，由消费方（动画系统）根据 VO 自行处理。

---

## VO Perspective 详解

`VO_1P` / `VO_3P` / `VO_1P_Critical` 是一个合成维度：

- **视角区分**：玩家自己听到的（1P） vs 其他人听到的（3P），可能有不同的音色、混响处理
- **播放优先级**：1P_Critical > 1P > 3P，决定多条 VO 同时触发时谁能打断谁

具体的行为配置（2D/3D、衰减、遮挡等）属于在音频中间件或引擎中处理的内容，待后续展开。
