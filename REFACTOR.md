# Game Audio Design 重构规划（v1）

本文档只记录两件事：

- 已达成共识的目标态（What）
- 接下来按顺序落地的任务（How）

长期规范与检查规则已迁移到 `README.md`，本文件不再重复规则细节。

---

## 1. 本次重构目标态（已定）

### 1.1 站点定位

- 个人专业文章站，不走“课程产品化”形态。
- 首页是唯一主入口，以“可持续写作”优先。

### 1.2 首页形态

- 视觉上无标题、无说明文案。
- 浏览器 `<title>` 固定为 `Game Audio Design`。
- 首页直接呈现：搜索框 + 全量文章流。
- 列表项只显示：标题 + 摘要（不显示日期文本）。
- 置顶策略：单流混排，最多 3 篇，使用 `[置顶]` 标签标识。
- 置顶排序：先按 `pinnedOrder`，重复时按 `date`。
- 非置顶排序：按 `date` 倒序。

### 1.3 导航与页面范围

- 不依赖导航栏。
- `docs/Home.md` 弃用，不再作为首页内容源。
- `Resource` 与 `Log` 作为站点入口下线，只保留文章主流。
- 文章页底部保留唯一出口：`返回首页`（同窗口打开）。
- 首页保留订阅模块，但弱化放置在底部。

### 1.4 链接与打开方式

- 首页文章链接：桌面新窗口、移动同窗口。
- 设备判定：宽度 + 指针能力（复合判断）。
- 搜索结果内链接行为与普通列表一致。
- 历史链接不做兼容（旧链接失效可接受）。

### 1.5 搜索交互

- 首页提供前端本地过滤搜索（标题 + 摘要）。
- 匹配规则：包含即命中（宽松匹配）。
- 搜索只过滤，不改变既有排序。
- 无占位文案，仅放大镜图标。
- 有输入时显示清空按钮；无输入不显示。
- 无结果时显示提示 + 一键清空。
- 显示计数：总数 / 当前匹配数。
- 快捷键：`/` 聚焦搜索框；`Esc` 先清空再失焦。
- 搜索状态不写入 URL 查询参数。
- 不做吸顶搜索框（移动端保持流式布局）。
- 搜索框保留 `aria-label`（视觉无变化）。

## 2. 内容模型（目标）

内容 frontmatter 以显式字段为主：

- `slug`: URL 标识（默认可由文件名转换得到，允许手动覆盖）
- `description`: 首页摘要与分享摘要来源
- `date`: 唯一时间字段（用于发布判定与排序）
- `pinned`: 是否候选置顶
- `pinnedOrder`: 置顶顺序

说明：

- `slug` 采用全小写 + 连字符风格。
- 页面可读标题来自正文，不绑定 `slug` 字段语义。
- `date` 与文末署名日期保持一致（执行规则见 `README.md`）。

## 3. 实施计划（按阶段推进）

## 阶段 A：数据与路由重塑

- [ ] 统一文章数据读取口径，首页改为自动拉取文章集合。
- [ ] 文章可见性与发布判定统一为 `date`（无 `date` 不发布）。
- [ ] 文章路由改为使用 `slug` 主键（支持手动覆盖）。
- [ ] 为冲突 `slug` 增加构建期失败保护。

## 阶段 B：首页重建（核心）

- [ ] 删除首页对 `docs/Home.md` 的依赖。
- [ ] 实现“置顶 3 篇 + 普通流”单流混排。
- [ ] 列表项输出“标题 + 摘要 + `[置顶]`”最小信息集。
- [ ] 底部保留弱化订阅模块。

## 阶段 C：搜索与交互

- [ ] 增加前端本地过滤搜索（标题 + 摘要）。
- [ ] 实现计数、无结果提示、清空按钮。
- [ ] 完成 `/` 与 `Esc` 键盘交互。
- [ ] 配置桌面/移动链接打开策略（含搜索结果）。

## 阶段 D：站点收口

- [ ] 下线 `Resource` 与 `Log` 入口。
- [ ] 文章页底部增加“返回首页”唯一出口（同窗口）。
- [ ] 清理已弃用的首页目录内容与相关文案。
- [ ] 对 RSS / sitemap 执行同一发布口径。

## 阶段 E：迁移与验收

- [ ] 为现有文章补齐 `slug` 与 `date`（按新规则）。
- [ ] 校验摘要质量与首页展示完整性。
- [ ] 全站回归：首页、文章页、RSS、sitemap、分享卡片。
- [ ] 上线后抽检桌面/移动端打开行为。

## 4. 风险与回退策略

- 风险：一次性切换首页数据源，可能导致链接缺项或排序异常。
  - 处理：先在本地构建全量校验，再发布。
- 风险：`slug` 批量迁移时发生重复。
  - 处理：保留构建失败机制，逐条修复后再上线。
- 风险：无 `date` 文档被误发布或误隐藏。
  - 处理：发布口径统一，以构建检查结果为唯一准入。
